
## MVP 스코프 및 개발 방향 정리
- 지난주 통합 업무 포털에서 검토했던 메일·보고서·회의 요약 등 부가 기능은 구현 범위에서 분리
- 멘토링 피드백을 반영하여, 기능을 한 번에 완성하기보다 업무 분장 핵심 흐름이 처음부터 끝까지 동작하는 MVP를 우선 구현하는 방향으로 재정립
- AI는 자동 배정이 아닌 담당자 후보·근거·위험 요소를 제시하고, PM이 최종 승인·수정·반려하는 의사결정 지원 방식으로 역할 명확화
- 핵심 MVP 구현 후 실제 동작 결과를 바탕으로 기능을 단계적으로 확장하는 개발 방식 확정

## 시스템 구조 및 Connector 설계 구체화
- 기존 통합형 플랫폼 구조에서, 기존 업무 시스템의 데이터를 필요한 범위만 연결하는 Connector 중심 구조로 재설계
- MVP 데이터 소스를 자체 구축 인력정보 DB, Jira, Google Drive로 구체화
    - 인력정보 DB: 조직·직무·스킬·근무시간·휴가 등 팀원 기준정보
    - Jira: 기존 업무·담당자·상태·일정·공수 등 업무량 정보
    - Google Drive: 기획서·요구사항·회의록 등 신규 업무 추출의 근거 문서
    - Google Calendar는 일정·가용시간 보정에 활용할 수 있도록 연동 방안을 검토하되, 1차 핵심 시연 이후 확장 기능으로 분리
- Connector별 수집 범위, 데이터 형식, 내부 데이터 모델 매핑, 인물 식별 기준 등 API 연동 요구사항 확인

## 비정형 문서 처리 및 데이터 모델 검토
- 프로젝트 문서에서 업무·일정·요구사항·제약 조건을 추출하기 위한 문서 처리 흐름 검토
- 문서 구조를 보존한 파싱, 청킹 전략, 메타데이터 설계, 업무·인력·문서 데이터를 연결하기 위한 Canonical Data Model 적용 방향 논의
- 데이터 수집 후 정규화, 동일 인물 매핑, 필수값 검증을 거쳐 업무 분장에 활용하는 데이터 흐름 정리
- 해당 설계는 MVP 구현에 맞춰 최소 수준으로 적용하며, 세부 방식은 구현 과정에서 확정 예정

## 기획·요구사항 문서 정합화 및 구현 준비
- 변경된 MVP 범위와 시스템 구조를 기준으로 기획서, 요구사항 정의서, WBS, 수집 데이터 보고서 간 내용 정합화
- 확정된 구조와 추가 검토 항목을 구분하여 팀 내 공유
- 다음 주부터 문서 기반 업무 추출, 업무량 분석, 담당자 추천·검증의 핵심 흐름 구현 착수 예정
