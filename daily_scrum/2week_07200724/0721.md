
스크럼 회의록

1. 멀티 에이전트 구조 재검토

진행 내용

* 전날 멘토링 이후 역할을 나누어 사용자 시나리오, 플로우차트, 시스템 아키텍처, 목업 등 기능 설계를 진행
* 기능을 문서로만 정의하는 과정에서 팀원마다 시스템 구조를 서로 다르게 이해하고 있다는 점을 확인
* 기존에 모든 기능을 멀티 에이전트로 구현하려는 방향으로 설계하면서 구조가 불필요하게 복잡해졌다는 문제를 인식

주요 논의

* 어떤 기능을 에이전트로 구현해야 하는지 재검토
* 멀티 에이전트 적용이 필요한 영역과 단순 워크플로우로 처리 가능한 영역 구분
* 기능 중심이 아닌 역할 중심으로 시스템을 다시 설계하기로 결정

⸻

2. 에이전트 · 워크플로우 · 툴 개념 정리

진행 내용

* 에이전트(Agent), 워크플로우(Workflow), 툴(Tool)의 개념과 역할을 다시 정의
* 각 요소가 시스템에서 담당하는 책임과 상호작용 방식을 정리

주요 논의

* 에이전트 : 목표를 달성하기 위해 판단 및 의사결정을 수행하는 역할
* 워크플로우 : 작업이 순차적으로 실행되는 전체 프로세스
* 툴 : 에이전트가 사용하는 외부 API 및 기능 단위

기대 효과

* 팀원 간 시스템 구조에 대한 공통된 이해 확보
* 이후 기능 설계 및 구현 과정에서 역할 혼선 최소화

⸻

3. 시스템 구조 재설계

진행 내용

* 사용자 시나리오를 작성하는 과정에서 기존 시스템 구조를 그대로 적용하기 어렵다는 점을 확인
* 시나리오 작성을 중단하고 시스템 구조를 우선적으로 재설계

변경 방향

* 기능 추가보다 전체 데이터 및 작업 흐름을 중심으로 구조 재정비
* 에이전트 간 역할과 데이터 흐름을 다시 정의
* 이후 사용자 시나리오와 플로우차트를 새로운 구조를 기준으로 작성하기로 결정

⸻

4. 목업 기반 워크플로우 검토

진행 내용

* Figma를 활용하여 화면 목업을 함께 작성
* 화면 흐름을 기준으로 사용자 동선과 워크플로우를 검토

논의 결과

* 문서만으로 논의할 때보다 시스템 흐름을 직관적으로 이해할 수 있었음
* 화면을 보며 의견을 공유하면서 팀원 간 구조에 대한 이해도가 크게 향상됨
* 사용자 경험(UX) 관점에서도 기능 흐름을 함께 검토할 수 있었음

⸻

금일 진행 결과

* 멀티 에이전트 구조를 재검토하고 설계 방향 수정
* 에이전트, 워크플로우, 툴의 역할 및 책임 정의
* 사용자 시나리오 작성 전 시스템 구조를 우선 재설계하기로 결정
* Figma 목업을 기반으로 워크플로우 및 화면 흐름 검토
* 팀원 간 시스템 구조에 대한 공통된 이해와 개발 방향 정립
