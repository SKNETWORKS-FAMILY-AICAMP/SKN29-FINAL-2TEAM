1. 프로젝트 주제 구체화

진행 내용

* 프로젝트 세부 주제를 AI 기반 프로젝트 업무 분석 및 인력 배정 코파일럿으로 구체화
* 서비스가 해결하고자 하는 문제와 핵심 기능을 명확히 정의

* 프로젝트 기획서를 기반으로 업무를 분석하고 작업(Task)을 도출
* 팀원의 업무량, 일정, 역량을 종합적으로 고려하여 담당자를 추천하는 방향으로 기능 범위 확정
* 향후 구현 범위를 고려하여 MVP 기능 중심으로 요구사항 정리

5. 업무 분배 데이터 요구사항 정의

진행 내용

업무 분배 기능 구현을 위한 필수 데이터와 처리 기준을 구체화

주요 정의

* Snapshot 객체별 필드 정의
* 필수 / 조건부 필수 / 선택 데이터 구분
* Google Docs 및 Google Calendar 활용 범위 정의
* 누락 데이터, 매핑 실패, 최신성(Freshness) 처리 기준 정리
* HOURS, POINTS, PROXY 계산 방식 정의
* Feature Readiness 판정 기준 초안 작성

⸻

현재 이슈

* Google Drive 및 Google Docs는 문서 기반 Task 추출 기능에서 조건부 필수 데이터로 활용
* Google Calendar는 일정 충돌 및 가용성 분석 시 조건부 필수 데이터로 활용
* Jira 사용자 매핑에 실패할 경우 해당 직원의 업무량은 계산하지 않는 방향으로 처리
* Feature Readiness 기준을 Figma, 기획서, 요구사항 정의서와 동일하게 동기화할 필요가 있음

⸻

내일 진행 예정

* Feature Readiness Check를 구현 가능한 수준까지 상세 설계
* 기능별 필수 입력 데이터와 SUCCESS / PARTIAL_RESULT / BLOCKED 판정 조건 확정
* 사용자 보완 요청 시나리오 및 오류 코드 정의
* Identity Mapping DB 및 API 계약 설계
* Snapshot 저장 구조 및 데이터 관리 방식 구체화
