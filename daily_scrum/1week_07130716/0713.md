# **아이디어 논의 정리**

## **최종 후보**

현재 프로젝트 방향을 아래 두 가지로 좁혀 논의

1. **문서 자동화 플랫폼**
2. **회의를 줄이는 AI 워크스페이스 플랫폼**

---

# **회의를 줄이는 AI 워크스페이스 방향**

## **핵심 목표**

회의를 대체하거나 최소화할 수 있도록, **팀 단위 워크스페이스에서 AI 에이전트가 업무를 분석하고 필요한 정보만 전달하는 플랫폼**을 구축

---

## **기본 구조**

- 회사 내 팀별 워크스페이스 제공
- 각 워크스페이스마다 여러 AI 에이전트 배치
- AI가 팀 내에서 발생하는 업무를 지속적으로 분석
- 필요한 경우에만 사람이 회의를 진행하도록 지원

예시

- 메일
- 메신저
- 메모
- 일정
- 작업(Task)

등의 정보를 하나의 워크스페이스에서 관리

---

## **에이전트 역할**

### **개인 에이전트**

- 개인 일정 관리
- 작업 진행 상황 분석
- 업무 우선순위 관리

### **팀 에이전트**

- 팀원의 업무 진행 상황 분석
- 일정 충돌 탐지
- 업무 병목 구간 분석
- 회의 필요 여부 판단

### **팀장 지원 에이전트**

팀장이 회의를 생성하면

- 현재 각 팀원의 진행 상황 분석
- 논의가 필요한 안건 추출
- 회의 Agenda 자동 생성
- 필요한 의사결정 및 아이디어 제안

→ 회의 전에 팀장이 핵심 내용만 확인할 수 있도록 지원

---

# **워크스페이스 구조**

플랫폼 하나 안에서

- 팀별 공간 생성
- 팀별 AI 비서 배치
- 팀별 업무 현황 관리

각 팀의 업무 특성은 달라도

공통 기능은 동일하게 제공

- 일정
- 메신저
- 메일
- 메모
- 작업 관리

---

# **도메인 적용 방식**

특정 산업에 맞춰 플랫폼을 개발하기보다

**플랫폼 구조는 범용으로 제공**

회사가 직접

- 업무 프로세스
- 팀별 규칙
- 업무 지침
- 회사 규정

등을 입력하여 사용할 수 있도록 구성

### **데모**

- 개발팀을 예시 도메인으로 시연
- 실제 서비스는 다양한 회사에서 사용할 수 있도록 설계

---

# **SLLM 학습 방향**

회사마다

- 업무 방식
- 용어
- 규정
- 업무 프로세스

가 모두 다르므로

플랫폼에서 관리자가

- 회사 지식
- 팀별 지침
- 업무 매뉴얼

등을 등록하여 학습할 수 있도록 설계

→ 회사별 맞춤형 AI Assistant 제공

---

# **반복 업무 자동화 방향**

핵심 목표

**매일 반복되는 업무를 AI가 대신 수행하도록 하는 것**

예시

- 일정 관리
- 진행 상황 공유
- 회의 내용 정리
- 메신저 내용 요약
- 업무 전달
- 공지 전달

# **논의 중 고민한 부분**

## **프로젝트 범위**

현재 아이디어가 너무 커질 가능성이 있음

→ 구현 범위를 어디까지 가져갈지 추가 논의 필요

---

## **회의를 어떻게 줄일 것인가**

단순 회의록 요약이 아니라

- 회의가 필요한지 판단
- 회의 전에 필요한 정보를 미리 제공
- 회의 후 자동으로 업무를 배분

등 실제 회의를 줄이는 방향으로 발전 필요

---

## **에이전트 구조**

현재 구조는

```
Agent A
    ↓
결과 생성

Agent B
    ↓
Agent A 결과 분석

Agent C
    ↓
Agent B 결과 분석
```

처럼

에이전트끼리 직접 대화하는 방식보다는

**이전 에이전트의 결과물을 입력으로 받아 다음 에이전트가 판단하는 파이프라인 형태**에 가까움.

→ 향후 에이전트 간 협업 구조를 어떻게 설계할지 추가 검토 필요
