# **1. AI 기반 업무 분장 및 재분배**

### **AI 기반 업무 분장**

- 할당했던 업무 패턴을 **Few-shot 형태로 학습**하여 자동 분류
- 직원의 **직무, 기술 스택, 역량 정보**를 기반으로 업무를 추천
    - 마이페이지에 기술 스택 태깅
    - 직원 프로필(이력서, 기술 스택 등)을 스캔하여 DB 구축
- 자연어로 업무를 분석하여 적합한 담당자에게 자동 분장
- 업무를 직원에게 할당할 때 **왜 해당 직원에게 배정했는지 근거를 함께 제시**

### **업무 재분배**

- 프로젝트 진행 중
    - 진행 속도가 느린 경우
    - 예상보다 빨리 끝난 경우
    - 업무 난이도가 예상과 다른 경우
- AI가 진행 상황을 판단하여 업무를 재분배
- 업무 분장이 잘못된 경우 재할당
- WBS 계획 대비 진행률을 계산하여 위험도 판단
    - 단순 진행률 입력은 거짓말이 가능하므로 AI가 중간 데이터를 기반으로 판단

※ 프로젝트 진행 후 한 달 정도의 데이터가 쌓이면 진행 속도, 피드백 등을 활용하는 기능은 **확장 기능**으로 고려

# **2. 직무 데이터 및 도메인 지식 활용**

- 업무 분장 시 **NCS(국가직무능력표준)** 활용 가능
    - 공공데이터 API 를 활용하여 JSON 형태
- NCS는 직무가 아닌 **교육 표준**이며,
    - “~할 수 있어야 한다.” 형태의 직무 능력 정의 제공
- 업무 분장 시 참고 데이터로 활용 가능

도메인 확장

- 특정 직종(예: 정보통신)만 대상으로 하지 말 것
- 플랫폼이므로 다양한 직무를 지원해야 함
- 처음에는 5개 직무로 개발하더라도
- 다른 직무를 추가해도 동일한 기능이 동작하도록 설계
- 하나의 직무로 충분히 검증한 후 확장

# **3. 멀티 에이전트 적용 기준**

모든 업무를 멀티 에이전트로 처리하는 것이 아니라

### **정형 업무**

- 단순 반복 업무
- 패턴이 일정한 업무

→ 일반 챗봇 또는 단일 AI로 처리 가능

### **비정형 업무**

- 불규칙한 데이터
- 상황 판단이 필요한 업무

→ 멀티 에이전트 적용

따라서 업무 유형을 먼저 판단하는 **전처리 과정**이 필요
