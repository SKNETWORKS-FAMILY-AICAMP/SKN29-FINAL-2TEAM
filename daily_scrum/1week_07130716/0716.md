- [x]  **프로젝트 핵심 문제**를 1~2문장으로 정리
    
    **[초안]**
    
    업무 분담 시 정확한 파악을 하기 어렵다
    
    **[다같이 다시 문제 정의]**
    
    - 이 업무가 얼마만큼의 부하가 생기는지 모름
        - 누구는 3일이 걸리고, 2일이 걸리고의 지표가 부족
    - 같은 직무일 때, 팀장이 생각한 직원의 평가지표가 필요함
    - 업무의 우선순위 부여는 팀장의 역할임
    
    이 지표들을 가지고 수치화할 지
    
    AI가 어떻게 판단할 것인가
    
    ### **최종 문제 정의**
    
    > 프로젝트에서는 구성원의 업무량과 역할을 객관적으로 파악하기 어려워 업무가 일부 인원에게 집중되거나 중복·누락되는 문제 발생
    > 

- [x]  **핵심 시나리오**(MVP) 하나 선정
    - 메일·메모·일정 다 설명하지 말고, "와우 포인트"가 될 대표 업무 흐름 하나만
    - 프로세스들을 문서화해서 우리 프로그램에 넣으면 그 프로세스로 업무 분담이 되게
    - 회사 프로세스 장기 기억으로 저장해야하는데 인풋이 뭐가 될건지?
    
    #### 업무량과 역할을 객관적으로 파악하여 업무 분배 제안
    
    #### 1) **업무 파악 및 세분화**
    
    팀에 주어진 목표를 달성하기 위해 필요한 전체 업무 리스트를 작성합니다.
    
    프로젝트를 더 작은 작업 단위로 세분화하고, 각 작업의 중요도, 난이도, 마감 기한을 정의합니다.
    
    #### 2) **구성원 역량 및 업무량 파악**
    
    팀원 개개인의 강점, 스킬, 현재 수행 중인 업무량(워크로드)을 정확하게 평가합니다.
    
    누군가에게 일이 과도하게 쏠리는 '병목 현상'을 방지하기 위해 각자의 여유 리소스를 확인합니다.
    
    #### 3) **적재적소 할당**
    
    파악된 업무의 난이도와 팀원의 역량을 매칭하여 역할을 부여합니다.
    
    역할과 책임을 명확히 설정하여 업무의 주도권과 최종 책임자가 누구인지 구성원 모두가 인지하도록 만듭니다.
    
    #### 4) **진행 상황 점검 및 피드백**
    
    업무가 배분된 후에도 정기적인 회의나 대시보드를 통해 진행 상황을 모니터링합니다.
    
    예기치 못한 이슈가 발생했을 때 업무를 재조정하거나, 완료 후 피드백을 통해 업무 방식을 개선하는 회고 과정을 거칩니다.
    
    현재 문제 정의를 기준으로 하면, \*\*메인 시나리오는 "업무를 배정하는 과정에서 업무량을 제대로 파악하지 못해 발생하는 문제를 AI가 해결하는 흐름"\*\*으로 구성하는 것이 가장 자연스럽습니다.
    
    ### 메인 시나리오
    
    ### 1) 프로젝트 생성
    
    ```jsx
    팀장이 새로운 프로젝트를 생성한다.
    - 프로젝트명 입력
    - 기획서 또는 요구사항 문서 업로드
    - 팀원 선택
    ```
    
    ↓
    
    ### 2) AI가 업무를 자동 추출
    
    ```jsx
    기획서를 분석하여 업무(Task)를 자동 생성한다.
    
    예)
    - 시장조사
    - 경쟁사 분석
    - UI 설계
    - API 개발
    - 테스트
    - 발표자료 작성
    ```
    
    ↓
    
    ### 3) 구성원 업무 현황 분석
    
    ```jsx
    AI가 기존 프로젝트 데이터를 분석하여
    각 구성원의
    
    - 현재 업무량
    - 진행 중인 업무
    - 예상 소요시간
    - 마감 일정
    - 가용시간
    - 역할
    - 역량
    
    을 자동 파악한다.
    
    **핵심 문제 해결**
    "현재 누가 얼마나 바쁜지 모르겠다."
    ```
    
    ↓
    
    ### 4) 업무 배정 추천
    
    ```jsx
    AI가 업무별 적합한 담당자를 추천한다.
    
    **추천 기준**
    - 현재 업무량
    - 가용시간
    - 역할
    - 기술 역량
    - 업무 난이도
    - 우선순위
    ```
    
    예시
    
    | 업무 | 추천 담당자 | 이유 |
    | --- | --- | --- |
    | 시장조사 | 김철수 | 현재 업무량 40%, 경험 多 |
    | API 개발 | 이영희 | 관련 프로젝트 경험 |
    | 발표자료 | 박민수 | 일정 여유 있음 |
    
    ↓
    
    ### 5) 팀장의 검토 및 수정
    
    ```jsx
    팀장은
    - 추천 결과 확인
    - 담당자 변경
    - 업무 추가
    - 업무 삭제
    
    후 확정한다.
    ```
    
    ↓
    
    ### 6) 프로젝트 진행
    
    ```jsx
    프로젝트가 진행되면서 AI는 지속적으로
    
    - 업무 완료
    - 일정 지연
    - 신규 업무 발생
    - 업무량 변화
    
    를 감지한다.
    ```
    
    ↓
    
    ### 7) 과부하 감지
    
    ```jsx
    김철수
    - 업무 8개
    - 예상 업무시간 52시간
    - 긴급 업무 4개
    
    ↓
    
    AI
    > "김철수의 업무량이 적정 수준을 초과했습니다."
    ```
    
    ↓
    
    ### 8) 업무 재배분 추천
    
    ```jsx
    AI
    
    시장조사
    → 박민수에게 이동 추천
    
    이유
    
    - 현재 업무량 25%
    - 동일 업무 경험
    - 일정 영향 없음
    
    ↓
    
    팀장이 승인하면 업무가 자동 재배정된다.
    ```
